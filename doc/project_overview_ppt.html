// problem

通用 AI 编程模型的
三个盲区

术语黑盒

"积分扣减"、"幂等键"、"GAM 回调"——企业术语对通用 LLM 只是字符串，无法映射到内部 SDK。

63% 开发者有此感受

返工率高

开发者平均需要 5 轮以上修改才能获得可用代码。签名过时、参数错误、漏掉配置。

5+ 轮/需求

规范缺失

模型不了解团队编码规范、历史缺陷模式、架构决策，生成的代码"能用但不合群"。

项目级理解缺失

目标：

AI 自动理解企业术语、API、规范与历史经验，在需求开发 / 问题定位 / 代码检视中提供精准生成与建议。

// architecture

Spec + RAG 双引擎

Spec 引擎

结构化规范契约作为硬约束，在生成前校验。接口契约 / 数据契约 / 行为规范 / 安全要求。

MUST obey

RAG 引擎

语义检索动态注入非结构化知识。历史方案 / API 文档 / 最佳实践 / 缺陷记录。
