# 私域知识赋能AI Coding工具 测试用例文档

## 1. 文档概述

### 1.1 测试范围
| 测试层面 | 覆盖模块 |
|---------|---------|
| 检索能力 | BM25检索、向量检索、图数据库检索、混合检索与重排序 |
| 需求开发场景 | 术语识别、历史方案参考、规范注入、Prompt组装 |
| 问题定位场景 | 错误触发、缺陷匹配、API变更检测、排查建议生成 |
| 代码检视场景 | Spec规范加载、命名规范检查、异常处理检查、代码差异解析 |
| MCP协议集成 | 工具暴露、上下文提取、工具调用响应、多轮对话上下文 |

### 1.2 测试环境要求
- **向量数据库**：Milvus / Qdrant（已配置并导入测试数据）
- **全文检索引擎**：Elasticsearch（已配置BM25索引）
- **图数据库**：Neo4j（已导入代码依赖图谱）
- **MCP Server**：已启动并可访问
- **测试客户端**：curl / Postman / MCP客户端SDK

## 2. 检索能力测试用例

### 2.1 BM25关键词检索

**用例编号**：RET-001  
**用例名称**：BM25关键词检索  
**测试类型**：集成测试  
**优先级**：P0  
**前置条件**：
- Elasticsearch中已存在测试索引`test_knowledge`
- 索引中包含术语"幂等性"、API名"IPaymentService"的文档

**测试步骤**：
```
1. 构造检索请求：keyword="幂等性"
2. 调用BM25检索接口
3. 记录返回的文档数量和相关性排序
4. 构造检索请求：keyword="IPaymentService.confirm"
5. 调用BM25检索接口
6. 记录返回结果
```

**预期结果**：
- 返回包含"幂等性"关键词的文档片段
- 关键词精确匹配的文档排序靠前
- 模糊匹配（如"幂等"）也能返回结果

### 2.4 混合检索与重排序

**用例编号**：RET-004  
**用例名称**：混合检索与重排序  
**测试类型**：集成测试  
**优先级**：P0  
**前置条件**：
- BM25、向量化、图数据库检索均可用
- Cross-BERT重排序模型已部署

**测试步骤**：
```
1. 构造查询："订单支付回调处理"
2. 同时发起BM25检索、向量检索、图数据库检索
3. 合并结果集，去重
4. 调用Cross-BERT重排序
5. 返回Top-5结果
```

**预期结果**：
- 三种检索结果正确合并
- 去重后无重复文档
- 重排序后最相关结果排在首位
- 召回率 > 92%

### 2.5 Top-K与阈值过滤

**用例编号**：RET-005  
**用例名称**：Top-K与阈值过滤  
**测试类型**：集成测试  
**优先级**：P1

- 检索接口支持Top-K和相似度阈值参数

**测试步骤**：
```
1. 设置Top-K=3，检索"支付接口"
2. 检查返回结果数量是否为3条
3. 设置相似度阈值=0.9，检索无关内容
4. 检查低相似度结果是否被过滤
```

**预期结果**：
- Top-K参数生效，返回结果数不超过K
- 低于阈值的结果被过滤，不返回
