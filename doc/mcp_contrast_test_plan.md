# AI Coding MCP能力对比测试方案

## 1. 文档概述

### 1.1 测试目的
通过对照实验，量化评估私域知识MCP能力对AI Coding工具代码生成质量的影响，为能力优化和ROI评估提供数据支撑。

### 1.2 实验设计
- **实验类型**：A/B对照实验（配对样本）
- **对照组**：无MCP，LLM直接响应用户需求
- **实验组**：有MCP，MCP注入私域知识后LLM生成代码
- **LLM选择**：同一模型，消除模型差异干扰
- **评审方式**：内部人工评审（盲审）

### 1.3 测试范围
| 场景 | 用例数 | 说明 |
|-----|-------|------|
| 需求开发-常规 | 8条 | 单一业务功能需求 |
| 需求开发-跨模块 | 6条 | 涉及多个API/模块协作 |
| 问题定位 | 8条 | 提供错误信息，验证修复建议 |
| 代码检视 | 5条 | 提供代码差异，验证规范检查 |

**总计**：27条测试用例

## 2. 测试指标体系

### 2.1 核心指标

| 指标名称 | 定义 | 评分方式 |
|---------|------|---------|
| **术语/API准确率** | 代码中使用的私域API、术语是否正确 | 评审人员检查SDK文档对照 |
| **规范符合率** | 符合团队Spec契约的程度 | 规范检查脚本 + 人工确认 |
| **上下文相关性** | 生成内容与用户需求的匹配程度 | 人工评分（1-5分） |
| **代码完整性** | 是否包含必要的边界处理、异常捕获等 | 人工评分（1-5分） |
| **采纳意向率** | 评审人员"愿意采纳此代码"的比例 | 二值判断（采纳/不采纳） |

### 2.2 辅助指标

| 指标名称 | 定义 | 数据来源 |
|---------|------|---------|
| **生成耗时** | 从请求到返回的响应时间 | 系统日志 |
| **Token消耗** | 单次请求消耗的Token数 | API调用记录 |
| **修改轮次** | 达到可用状态需要的交互次数 | 对话记录分析 |

### 2.3 评分标准

| 分值 | 含义 |
|-----|------|
| 5分 | 优秀，完全符合预期，无需修改 |
| 4分 | 良好，有小瑕疵但易于修正 |
| 3分 | 一般，需要较多修改才能使用 |
| 2分 | 较差，偏离预期较多 |
| 1分 | 差，完全不可用 |

## 4. 测试执行流程

### 4.1 准备阶段

```
[Step 1] 准备测试环境
├── 部署MCP Server并确认可用
├── 准备测试知识库（导入测试数据）
├── 确认LLM API可用并记录模型版本
└── 验证IDE测试通道连接

[Step 2] 准备评审材料
├── 打印/导出全部27条用例
├── 准备评分卡（电子表格）
├── 安排评审人员（3人）
└── 盲审规则说明

[Step 3] 生成对照代码
├── 对每条用例执行：无MCP生成
├── 对每条用例执行：有MCP生成
├── 随机打乱顺序，编号A/B版
└── 记录每次生成的耗时和Token消耗
```

### 4.2 执行阶段

```
[Step 4] 评审打分
├── 评审人员对A/B版本独立打分
├── 每条用例填写评分卡
├── 不标注哪个是A/B版本
└── 记录修改建议

[Step 5] 数据汇总
├── 收集所有评分卡
├── 计算各维度平均分
├── 统计采纳意向率
└── 整理典型案例
```

### 4.3 分析阶段

```
[Step 6] 统计分析
├── 配对样本t检验（α=0.05）
├── 计算效应量（Cohen's d）
├── 绘制雷达图/柱状图对比
└── 撰写分析报告
```

## 6. 统计方法

### 6.1 配对样本t检验

**原假设H0**：有MCP与无MCP的生成质量无显著差异  
**备择假设H1**：有MCP的生成质量显著优于无MCP

**检验公式**：
```
t = μ_diff / (σ_diff / √n)
```

**显著性水平**：α = 0.05  
**判断标准**：p < 0.05 时拒绝H0，认为差异显著

### 6.2 效应量计算

**Cohen's d**：
```
d = (μ_with_mcp - μ_without_mcp) / σ_pooled
```

| d值 | 效应大小 |
|-----|---------|
| 0.2 | 小效应 |
| 0.5 | 中效应 |
| 0.8 | 大效应 |

### 6.3 场景分层分析

分别对四个场景计算：
- 各场景平均分对比
- 各场景采纳率对比
- 各场景典型案例标注

## 9. 风险与应对

| 风险 | 影响 | 应对措施 |
|-----|------|---------|
| 评审人员主观偏差 | 评分不客观 | 盲审、3人独立评审取平均 |
| 测试用例代表性不足 | 结论泛化能力弱 | 用例覆盖核心场景，允许补充 |
| LLM输出波动 | 同一输入多次结果不同 | 固定temperature=0.3，多次取样 |
| 知识库数据不足 | MCP注入内容不完整 | 预填充测试知识，确保覆盖率 |
